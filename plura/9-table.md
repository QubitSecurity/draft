## 9. 보안성 비교표

*(로그 무결성, 전송 암호화, 위변조 방지 기능 등)*

| 구분                      | **PLURA-XDR**                                          | 타제품 A             | 타제품 B              |
| ----------------------- | ------------------------------------------------------ | ----------------- | ------------------ |
| **로그 무결성 보장**           | ✔ 수집 시점부터 해시 기반 무결성 검증<br>✔ 저장 로그에 대한 무결성 점검 기능 제공     | △ 저장 시 단순 해시만 적용  | × 무결성 검증 기능 없음     |
| **전송 구간 암호화**           | ✔ 에이전트 ↔ 서버, 서버 ↔ 연동시스템 간 **TLS 암호화** 기본 적용            | △ 일부 구간만 암호화      | △ 옵션 또는 제한적 지원     |
| **로그 위변조 방지 기능**        | ✔ 운영자도 원본 로그 삭제·수정 불가 구조<br>✔ 삭제·변경 시도 자체를 감사로그로 기록    | △ 삭제 로그만 일부 기록    | × 위변조 방지 설계 미흡     |
| **관리자·운영자 접근통제**        | ✔ RBAC(역할 기반 권한관리) 지원<br>✔ 관리자·운영자·조회자 등 권한 세분화        | △ 기본 관리자/사용자 2단계  | △ 단일 관리자 계정 중심     |
| **관리자 감사로그(Audit Log)** | ✔ 설정 변경, 정책 수정, 로그인 등 **모든 관리자 행위 로그** 남김              | △ 일부 주요 이벤트만 기록   | × 감사 기능 미흡         |
| **정책·탐지 룰 무결성 관리**      | ✔ 정책/필터 변경 시 이력·버전 관리 자동 기록                            | △ 최신 변경 내역만 확인 가능 | × 변경 이력 관리 기능 없음   |
| **저장 데이터 보호(접근제어)**     | ✔ 중요 로그·이벤트에 대한 DB·파일 접근권한 제한<br>✔ 내부자에 의한 직접 조회·수정 차단 | △ DB 권한 위주        | △ 파일 단위 통제 미흡      |
| **계정/인증 보안**            | ✔ 관리자 계정에 강력한 패스워드·MFA 적용 가능                           | △ MFA 일부 지원       | × 기본 인증(아이디/비번) 수준 |
| **보안 설정 가이드 및 운영 규범**   | ✔ 로그·계정·권한·암호화 관련 보안 가이드 제공                            | △ 기본 설치 가이드 수준    | × 별도 보안 가이드 부재     |
