

# ■ ③ 제품 보안성 비교표

(로그 위변조 방지, 전송 암호화, 무결성 보장 등)

| 구분                  | **PLURA-XDR**                                                                      | 타제품 A              | 타제품 B             |
| ------------------- | ---------------------------------------------------------------------------------- | ------------------ | ----------------- |
| **로그 위변조 방지 기술**    | ✔ 에이전트 단·서버 단 모두 **위변조 방지(Integrity Seal)** 적용<br>✔ 서버/PC에서 수집 즉시 서명 및 변조 여부 자동 검증 | △ 일부 로그만 해시 처리     | × 위변조 방지 기능 부재    |
| **전송 구간 암호화**       | ✔ 모든 통신(TCP/HTTPS) **TLS 기반 암호화** 적용<br>✔ 서버–Agent–SIEM 간 End-to-End 암호화           | △ 일부 구간 암호화        | △ 옵션 기능 또는 제한적 지원 |
| **저장 데이터 무결성 보장**   | ✔ 이벤트·로그 저장 시 **Hash Chain 기반 무결성 보장**<br>✔ tamper-proof 로그 아키텍처 제공                | △ 저장 시 단순 해시       | × 저장 무결성 미지원      |
| **권한 관리 및 접근통제**    | ✔ RBAC(역할 기반 접근 제어) 지원<br>✔ 운영자·사용자·보안담당 역할 구분 및 세분화                               | △ 역할 관리 단순         | × 운영자 권한 제어 미흡    |
| **계정 및 인증 보안**      | ✔ MFA(다중 인증) 적용 가능<br>✔ 관리자 계정 보호 및 비정상 접근 차단                                      | △ MFA 일부 지원        | × 기본 인증 방식 중심     |
| **로그 삭제/변경 통제**     | ✔ 운영자라도 **원본 로그 삭제·변경 불가** 구조<br>✔ 삭제 이벤트도 별도 감사 로그로 남김                            | △ 삭제 로그 일부만 기록     | × 삭제/변경 제한 기능 없음  |
| **보안 정책·필터 무결성 보호** | ✔ 탐지 정책 변경 시 **감사이력·버전관리 자동 기록**                                                   | △ 일부 정책만 버전 관리     | × 정책 무결성 검증 미지원   |
| **탐지 엔진 보안성**       | ✔ MITRE 기반 TTP 매칭 + 악성행위 정밀 분석<br>✔ Unknown Attack 탐지(비정형 행위 탐지)                   | △ 룰 기반 위주          | △ 특정 위협만 탐지       |
| **관리자 감사 로그**       | ✔ 모든 관리자 활동(설정, 정책 변경, 로그인 등) 기록<br>✔ 비인가 접근 및 관리자 오남용 탐지 가능                       | △ 로그인 위주 기록        | × 관리자 감사 기능 부족    |
| **에이전트 보안성**        | ✔ 경량·비침습 Agent, OS 안정성 보장<br>✔ 악성코드 위장/조작에 강한 자체 검증 로직                             | △ 자원 사용량 높고 충돌 가능성 | △ 특정 OS만 지원       |
