
# ■ 8. 규격·확장성 비교표

*(HW 구성, SW 구조, 확장 가능 용량, 향후 증설 방식 등)*

| 구분                      | **PLURA-XDR**                                                                      | 타제품 A                | 타제품 B              |
| ----------------------- | ---------------------------------------------------------------------------------- | -------------------- | ------------------ |
| **HW 구성**               | ✔ HW·SW 일체형 어플라이언스 구성 가능<br>✔ 고성능 스토리지·멀티코어 기반 고가용성 설계<br>✔ 단일 장비로 수집·정규화·탐지·대응 처리 | △ 서버· 스토리지 별도 구성 필요  | △ 추가 장비 의존도 높음     |
| **SW 구조**               | ✔ 통합 XDR 엔진(수집→정규화→탐지→차단) 단일 플랫폼 제공<br>✔ 에이전트 기반 서버·PC·웹 패킷 통합 구조                  | △ 모듈별 분리 구조          | △ 타사 제품 연동 필요      |
| **확장 가능 용량(Scale-out)** | ✔ 서버/스토리지 수평 확장으로 처리량 증가 가능<br>✔ 단말 증가·로그 증가 시 자동 분산 처리 구조                         | △ 일정 용량 이상 확장 한계     | △ 확장 시 별도 제품 구매 필요 |
| **처리 용량 (일 로그 처리량)**    | ✔ **100GB/일 이상** 기본 설계<br>✔ 다중 노드 구성 시 TB급까지 확장 가능                                 | △ 20~50GB/일 수준       | △ 확장 시 장애 위험 증가    |
| **탐지 엔진 확장성**           | ✔ 탐지 정책 수백~수천 개 적용 시 성능 저하 없음                                                      | △ 필터 증가 시 탐지 지연      | △ 정책 제한 존재         |
| **단말 확장성**              | ✔ 서버 200대 + PC 10,000대 환경 대응<br>✔ 단말 추가 시 라이선스만으로 확장 용이                            | △ 2천~3천 단말 수준        | △ 대규모 단말 미지원       |
| **연동 확장성**              | ✔ Syslog·CEF·JSON·API 등 다양한 연동 지원                                                  | △ 일부 Syslog 중심       | △ 연동 표준 부재         |
| **HW 증설 방식**            | ✔ 성능·용량 부족 시 **노드 추가(Scale-out)** 방식으로 무중단 확장                                      | △ 장비 교체(Scale-up) 필요 | △ 증설 시 서비스 중단 가능성  |
| **SW 증설 방식**            | ✔ 모듈 기반 확장(분석·정규화·탐지 엔진 분리 가능)                                                     | △ 모놀리식 구조로 유연성 낮음    | △ 별도 제품 연계 필요      |
| **고가용성(HA) 구성**         | ✔ Active-Standby 및 Active-Active 구성 지원                                             | △ HA 지원 제약적          | △ 별도 옵션 필요         |
